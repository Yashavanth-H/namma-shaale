# 🏫 Namma Shaale: AI-Powered Inventory Management

**Namma Shaale** (Our School) is a production-grade Android application designed specifically for Karnataka's schools to manage their physical assets with the power of Artificial Intelligence. Built with Jetpack Compose and integrated with Gemini AI via OpenRouter, it transforms inventory tracking from a chore into an intelligent, data-driven process.

![Namma Shaale Banner](https://images.unsplash.com/photo-1509062522246-3755977927d7?q=80&w=1200&auto=format&fit=crop)

## ✨ Key Features

### 🤖 Smart AI Assistant
*   **Audit Summaries**: Generates concise, professional reports for school principals based on audit data.
*   **Maintenance Predictions**: Analyzes asset history to predict when a computer, desk, or lab equipment might need repair.
*   **Smart Repair Insights**: Provides technical suggestions for reported issues to help non-technical staff.

### 📋 Inventory Management
*   **Asset Tracking**: Manage Electronics, Furniture, Sports Equipment, and more.
*   **Condition Monitoring**: Real-time status updates (Working, Needs Repair, Broken, Missing).
*   **Camera Integration**: Attach photos to assets directly during the intake process.

### 📊 Reporting & Audits
*   **Instant Audits**: Conduct school-wide audits in minutes.
*   **CSV Exports**: Export your entire inventory for official record-keeping.
*   **Digital History**: Every repair and audit is logged and searchable.

## 🛠 Tech Stack
*   **UI**: Jetpack Compose (Modern Declarative UI)
*   **Architecture**: MVVM with ViewModel and StateFlow
*   **Database**: Room (Local Persistence)
*   **AI**: OpenRouter API (Gemini 2.0 Flash)
*   **Networking**: OkHttp
*   **Image Loading**: Coil

## 🚀 Getting Started

### Prerequisites
*   Android Studio Ladybug or later
*   OpenRouter API Key

### Installation
1.  Clone the repository:
    ```bash
    git clone https://github.com/Yashavanth-H/namma-shaale.git
    ```
2.  Add your API key to `local.properties`:
    ```properties
    openrouter.api.key=YOUR_API_KEY_HERE
    ```
3.  Build and run on your device or emulator.
